# Home Manager

This directory contains the user-space half of MD4N.
If `nixos/` defines the machine, `home-manager/` defines the everyday environment that lands in the user's home directory: shell behavior, application config, desktop entry overrides, wallpapers, and profile-aware package selections.

## What Lives Here

- `home.nix`
  Stable Home Manager entrypoint used by the flake.
  It stays intentionally small and mostly wires together the internal modules based on `user.packageProfile` and the generated local settings.
- `modules/`
  Reusable Home Manager modules split by responsibility such as core wiring, programs, services, fonts, and profile-specific package sets.
- `config/`
  Shared application config that is safe to commit.
  This includes Niri, Fish, Fcitx5, Noctalia, Yazi, Alacritty, and other user-facing tools.
- `applications/`
  `.desktop` overrides installed under `~/.local/share/applications`.
  Use this when the upstream launcher file needs better defaults or custom launch behavior.
- `Wallpapers/`
  Shared wallpaper assets that should be installed for the user.

## How It Is Used

`flake.nix` exports a Home Manager configuration that points at `home.nix`.
From there, the modules in this directory link repository-managed files into `~/.config`, `~/.local/share`, and other standard XDG locations.
Some paths intentionally point to generated machine-local files such as the Niri outputs file and browser launcher script, but those generated files live outside this directory.

## What Does Not Belong Here

- Secrets
- Host-specific runtime files that are regenerated by setup
- Throwaway local experiments that are not meant to be shared across machines

As a rule of thumb, if another machine using MD4N should get the same file, it belongs here. If it only exists because of one machine's hardware or one user's local state, it should stay outside the repository.
