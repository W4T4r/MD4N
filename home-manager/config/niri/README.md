# Niri

Shared Niri configuration tracked in Git.
The whole directory is linked directly from the repository into `~/.config/niri`.
`outputs.local.kdl` is generated locally into `~/.config/niri/` so display changes can be tested in place while the shared Niri tree stays linked.
`browser.sh` is generated locally into `~/.config/md4n/generated/niri/` and is called with an absolute path.
